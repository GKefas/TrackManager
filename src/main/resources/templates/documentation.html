TrackManager API Documentation v1

This is the documentation for the Track Manager API. Below are the available endpoints and their descriptions:

Pagination & Query Parameters

All the endpoints that support listing (GET requests) support pagination and optional filtering. You can use some of the following query parameters based on endpoint:

  • page: The page number for pagination (default is 1, which represents the first page).
  • size: The number of items per page default is 10 if u only provided page otherwise it will retrieve all data.
  • title: Filter albums or tracks by title (optional).
  • artistName: Filter albums or tracks by artist name (optional).
  • name: Filter artists or genres by name (optional).

For example, to retrieve the first page of albums sorted by title in ascending order, you would use the following URL:

/api/albums?page=1&size=10
Endpoint Method Description Response
/api/albums GET Retrieves all albums or filters albums by title/artistName. List of AlbumDTOs (id, title, artistName)
/api/albums/{id} GET Retrieves an album by its unique ID. AlbumDTO (id, title, artistName)
/api/artists GET Retrieves all artists or filters by name. List of ArtistDTOs (id, name)
/api/artists/{id} GET Retrieves an artist by their unique ID with tracks. ArtistDTO (id, name, tracks)
/api/genre GET Retrieves all genres. List of Genre (id, name)
/api/mediaType GET Retrieves all media types. List of MediaType (id, name)
/api/tracks GET Retrieves all tracks or filters tracks by parameters. List of TrackDTOs (id, title, duration)
/api/tracks/{id} GET Retrieves a track by its unique ID. TrackDTO (id, title, duration)
